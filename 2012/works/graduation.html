Overview

Because lots of students doing their graduation project at home independently, back to university to meet their professors and deliver their paper will waste a lot of time. Therefore, I chose a graduation project, which named Graduation Project Management System (GPMS) to attempt to solve this problem. Including establish projects for professors, choose projects and deliver papers for students, etc., the GPMS can gives both of them great convenience during graduation project.

Process

At the beginning, I wrote a requirements document and discussed with my advisor, Zhenlian Shi, to determine which features should the system includes and how to implement it. I drew some sketches to help me express my idea: a conspicuous timeline was showed in any page. My advisor thought it was a cool idea, so I started to design the interface subsequently.

Designing the website and implementing it by HTML and CSS is easy for me. Background programming, however, was the greatest challenge. Thanks to the OA system project we had finished in September, I do have a little experience about programming with Java and SQL. So I could learn from it and figured out this challenge.

I was very interesting in design, so I paid more attention on the details of interface and interaction, e.g. every button had different look when users' pointer hover it, lots of input boxes had grey hint in the boxes and plenty of mini icons to help users to learn how to use quickly. As a result, my project was very pretty ( I think, :) ) and user friendly.
